- Use **TDD** (test-driven-development)
   1. Write the test first. Each test must match a given behavior. We are not interested in testing implementation details.
   2. **Implement** the simplest version that satisfies the test.
   3. **Refactor** to clean up design-smells and make the implementation more idiomatic.
   4. Make sure tests still pass.
   5. Repeat
- All 3rd party dependencies must be explicitly approved by a human.
